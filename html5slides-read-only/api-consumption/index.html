What is a API?

An application programming interface (API) is a specification intended to be used as an interface by software components to communicate with each other.

Source: Wikipedia

You can actually make an api with Javascript as well. Build your API with Node.js

Tap into an Api

Api can be called throught diffent http request methods types (eg : post, get, put, delete), so as long as you can send a request to the api you can get infomation from it. Frontend api consumption is pretty limited compared to server side consumption.

I reccomend starting with consuming JSON formatted APIS first. Its easier and fomatted for Javascript.

pure JS example

var get = function(url, callback){

    var xmlhttp = new XMLHttpRequest();
    // Open a connection to the server
    xmlhttp.open("GET", url, true);

    // Setup a function for the server to run when it's done
    xmlhttp.onreadystatechange = callback;

    // Send the request
    xmlhttp.send(null);
  };

get('/path-to/test.json', function(e){

  console.log(this.responseText);
  
})

`xmlHttp` is window object for AJAX calls and `updateInfo` is a function that we can pass the data to.

jQuery example

To use this example you will need to have jQuery injected into your page.

$.getJSON(‘/path-to-api/filename.json’, updateInfo); // shorthand jquery has many methods for ajax

`getJSON` is a shorthand method for AJAX calls in jquery, we are also passing the data to updateInfo.

Local vs Remote Calls

This is where the whole server / client side issues come in. When working locally you can use any type of api, and pull the information into server side or client side. Also I mean local to the application, so if you place the application on a remote server, that remote server would be local to that application and it can make all the call locally.Eg of a client side local call.

$.ajax({
  url : ‘/test.json’, // local path
  dataType : ‘json’, //json data type
  method : ‘get’, // http method
  success : function(response){ // callback for success
    console.log(response.success); // will log out true              
  }
});

Remote Calls Client Side

Remote calls from client side are a bit differnt, browsers will not allow you to get data out of an remote api unless it is allowed to. Then api owner needs to enable thorugh cross domain policies. An easy way around this for JSON apis is to use JSONP calls which is a ‘JSON with Padding’. The padding allows you to pass the data to a function in javascript.

$.ajax({
  url       : ‘http://example.com/test.json’, // local path
  dataType  : ‘jsonp’, //jsonp data type
  method    : ‘get’, // http method
  data      : {jsonp : ‘?’}, // specific to jquery 
  success   : function(response){ // callback for success
    console.log(response.success); // will log out true              
  }
});
